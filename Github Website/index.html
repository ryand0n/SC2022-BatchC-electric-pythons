What Is Our MVP?

This website takes user input data and returns estimated salary as STEM worker.

Why Use Our Product?

We attempt to make estimating your salary effortless based on your experience, education level, and specific career in STEM fields. This can be useful to professionals in determining career prospects, as well as those looking into STEM as a profession, to model a possible salary.

Linear Regression Model
The linear regression model shows the linear relationship between the x-variable, the input data and the y-variable, which in this case is the total yearly compensation of the STEM workers within the database. This essentially finds the line that best fits the data points used to model a target prediction value.
Nearest Neighbors Regression Model
KNN uses “feature similarity” in the input data to predict the value of any new input data, assigning it based on how similar the new data point is to points in the training set. The similarity is usually calculated with distance functions.
Decision Trees Regression Model
The Decision Tree method used for regression predicts the value of a target variable by breaking down a dataset into subsets to organize the data into different nodes which gets incrementally developed.

Data Science Pipeline

the set of processes that convert raw data into actionable answers

  • Step 1

    Finding the Dataset

    Our journey to find this dataset began on Kaggle. We searched Kaggle to find the appropriate dataset, going through multiple options in different fields before settling on the Data Science Salary set. Once we'd decided on our project, we downloaded that dataset and got to work!

  • Step 2

    Data Cleaning

    To begin with, we removed the columns that included data that we couldn’t use or didn’t need such as dmaid, city id, timestamp, rowNumber, tag, and other details. We then removed the rows with NA variables. We also removed the rows that had no information on race, education, or basesalary. We found some rows with outliers or incorrect information that we took out of our dataset as well. Finally, we simplified the information on location, education, and race.

  • />

    Step 3

    Making Visualizations

    We took our cleaned data and, using pandas and plotly, created several awesome visualizations of the data. These included scatterplots, histograms, bar graphs, and pie charts.

  • Step 4

    One Hot Encoding

    One hot encoding is one method of converting data to prepare it for an algorithm and get a better prediction. With one-hot, we convert each categorical value into a new categorical column and assign a binary value of 1 or 0 to those columns. Each integer value is represented as a binary vector. The way this applies to our model is that it helps the AI understand for example what degree someone has.

  • Step 5

    Train Test Split

    The train test split method is used to estimate the performance of machine learning algorithms. This method works by comparing data in our machine learning model to machine results. We split our test data into 20% of the actual data and the training set is split into 80% of the actual data. We let our model use our testing inputs to make predictions on the total yearly compensation (gross income on an annual basis).

  • Step 6

    Creating ML Models

    Using the train and test sets, we trained three machine learning models on the data: Linear Regression, Nearest Neighbors Regression, and Decision Trees Regression. The accuracy rate for the Linear Regression model was the highest at 92.57%, making this model the most successful.

  • Step 7

    GridSearch CV

    We used GridSearchCV to find the best parameters for our three ML models to return the most accurate results for a given model.
